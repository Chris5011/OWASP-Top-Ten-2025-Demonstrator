Teaching Point

LDAP injection follows the same general injection pattern: user input crosses from directory attribute value into LDAP filter logic.

Unsafe

(&(uid=USER_INPUT)(userPassword=USER_INPUT))

Safer

(&(uid=escape(USER_INPUT))(userPassword=escape(USER_INPUT)))
