SQL Injection: Login Example

SQL injection is one of the oldest and most impactful vulnerabilities in web security. It occurs when an application builds a SQL query by concatenating strings that include unvalidated user input. The database engine cannot distinguish between the developer's intent and the attacker's inserted logic — it simply executes whatever query it receives.

How It Works

1. Developer writes a template

"SELECT * FROM users WHERE username = '" + username + "' AND password = '" + password + "'"

2. Attacker submits: admin' OR '1'='1

SELECT * FROM users WHERE username = 'admin' OR '1'='1' AND password = 'anything'

The condition '1'='1' is always true, so the WHERE clause matches every row. The database returns the first user — typically the admin account.

Comment injection: admin' --

SELECT * FROM users WHERE username = 'admin' --' AND password = '...'

The double-dash -- starts a SQL comment, discarding everything after it. The password check is silently removed from the query.

Safe: parameterized query

SELECT * FROM users WHERE username = ? AND password = ?

bind("admin' OR '1'='1", "anything")

The query template is compiled first. The user input is then passed as a typed value, never interpreted as SQL syntax.

Real-World Impact

What an attacker can achieve

  • Bypass login without knowing any password
  • Dump all usernames, passwords, and email addresses
  • Read arbitrary database tables (UNION SELECT)
  • Modify or delete data
  • In some configurations: read files from the server or execute OS commands

Correct fix

  • Always use parameterized queries or prepared statements
  • ORM frameworks (SQLAlchemy, Hibernate, ActiveRecord) do this by default when used correctly
  • Input validation is useful but is not a substitute — parameterize first
  • Apply least privilege: the DB user should not have DROP or FILE permissions

Teaching Point

Wrong mental model

"I will filter out the bad characters."

Blocklist filtering is fragile — there are always edge cases, encoding tricks, or characters you missed.

Correct mental model

"Data and instructions must travel through separate channels."

Parameterized queries enforce this separation structurally — the driver makes injection impossible by construction.
