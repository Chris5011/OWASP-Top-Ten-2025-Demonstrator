5. OWASP Focus

Authentication Failure

The application treats decoded JWT claims as authenticated identity before proving that the token is valid.

decode(token) → trust(user, role)

Correct Flow

The application validates the token first, then creates the authenticated user context.

verify(token) → decode(claims) → authenticate

Teaching Point

Wrong

“JWTs are signed, so decoding them is enough.”

Better

“Authentication starts after verification, not before.”
