3. Attack Explanation

Session Fixation

The attacker forces or predicts a session ID before login. If the application keeps that ID after login, the attacker can reuse it.

known_session_id → victim_login → attacker_reuse

Session Hijacking

The attacker steals an already authenticated session ID. If the session is still valid, authentication is bypassed completely.

stolen_session_id → authenticated_request
