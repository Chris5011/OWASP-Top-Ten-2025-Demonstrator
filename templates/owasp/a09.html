A09: Security Logging & Alerting Failures

Even a perfectly secured application can be compromised eventually. When that happens, the ability to detect, investigate, and respond to an attack depends entirely on the quality of your logs. Without adequate logging, attackers can operate undetected for weeks or months. According to industry reports, the average dwell time — the period between breach and detection — is over 200 days in organizations with poor logging practices.

What Makes a Log Security-Useful?

Poor Log

User login failed
User login failed
User login successful
  • No timestamp — when did this happen?
  • No identity — which user?
  • No source — which IP address?
  • No correlation — are these related events?
  • Unstructured text — hard to query or alert on

Good Log (structured)

{"ts":"10:00:01","event":"LOGIN_FAIL","user":"admin","ip":"192.168.1.50","attempt":1}
{"ts":"10:00:03","event":"LOGIN_FAIL","user":"admin","ip":"192.168.1.50","attempt":2}
{"ts":"10:00:05","event":"LOGIN_FAIL","user":"admin","ip":"192.168.1.50","attempt":3}
{"ts":"10:00:10","event":"LOGIN_SUCCESS","user":"admin","ip":"192.168.1.50"}
{"ts":"10:00:10","event":"ALERT","rule":"BRUTE_FORCE","user":"admin","ip":"192.168.1.50"}
  • Timestamped, attributed, and machine-readable
  • Correlated across events for the same user and IP
  • Alert fired automatically based on a detection rule

What Should Be Logged?

Authentication & Access

  • Login success and failure (with username, IP, timestamp)
  • Password reset requests
  • MFA failures
  • Access to sensitive resources
  • Admin actions (role changes, user creation/deletion)

Application Events

  • Input validation failures (especially repeated ones)
  • Exceptions and unexpected error conditions
  • Data export or bulk read operations
  • Cryptographic failures
  • Rate limit violations

What NOT to log (data to never record)

Passwords and password hashes, session tokens, credit card numbers, private keys, or any PII beyond what is necessary for investigation. Logs are often stored less securely than databases — attacker access to logs should not yield additional credentials.

Teaching Point

Logging is not just an operational concern — it is a security control. Without structured, correlated, and alerting-capable logs, attackers operate in the dark. Every security incident eventually becomes a logging and forensics problem.
